## F. What Should Not Be Changed Before Sharing

Before sharing this project publicly, avoid unnecessary changes that may introduce instability.

Do not:

- Add new features before sharing the current stable MVP.
- Change `app.py` unless there is a real product or bug requirement.
- Change `core/` or `config/` unless there is a confirmed bug.
- Add paid APIs.
- Add features from unrelated projects.
- Add credentials, tokens, passwords, secrets, or private data.
- Mix this project with unrelated codebases.

## G. Sharing Checklist

Before sharing the project, check the following:
